JavaScript 高级程序设计之基本概念

Posted on 2017-04-18

ADDVICE

  • 通过添加括号来减少开发过程中的复杂性, 增加可阅读性;
  • 当创建对象的属性时, 可以使用 null 来赋值;
  • 可以使用 x === undefined 来检测某个值或函数的某个参数是否存在
  • 检测某个东西或某个值是否存在或是否为空 x == null
  • 在平常开发时不要使用连等操作符( a = b =c)

JS数据类型

基础数据类型(不可改变)

  • number
    • 无穷
    • NaN 不等于任何数字的数字 NaN ≠ NaN
    • 常规数字
      • 整数尽量使用10进制,少用16进制,不用8进制
      • 小数
        • 0.1 + 0.2 ≠ 0.3 原因在于 JS 使用二进制来表示数字. 而且 JS 解释器中只能
        • 若要得到精确的结果
          • 保留两位小数
          • 在比价或计算时先化为整数,计算完毕后再转换回来.
  • boolean
    • 只有下述的内容是 false;
      1. undefined
      2. 0
      3. -0
      4. NaN
      5. 空的字符串
      6. null
    • 所有对象包括空对象都是 true;
  • null, 表示一个空对象指针[:不是很理解]
  • undefined

  • string 字符串

    • 本质是 Unicode 序列, 因此不管什么语言只要能通过一个 Unicode 来表达,那长度(length)就是1;

    • string.length() 能使用的原因在于 JS 的解释器遇到.或[]时,运行规则是首先判断前面表示式是否为对象,如果不是则转化为对象.

null 与 undefined 的异同

  1. null 表示一个值被定义为”空值”; undefined 表示根本不存在该定义.

    • 可以设置一个值为 null . 但设置 undefined 是不合理的, 而是要使用delete; - 判断一个定义是否存在的时候应该使用 undefined 而不是 null

      这个概念在 JSON 规范中被强化, 其中不存在 undefined 这个类型, 但存在 null. 同时一些库(如 jQuery) 中的深度拷贝函数会忽略 undefined 而不会忽略 null

  • null 是关键字
  • null 本质是空的对象, undefined 是 windows 的属性, 这个属性的值叫 undefined
  • typeof 时 ,null -> object , undefined -> undefined. 由第二点可见.
  • 数字转换时 null -> 0, undefined->NaN

null 与 undefined 的共同点

  • 在基础数据类型中都只有一个值;
  • 因为它们不能被转化为对象(否则可以有 length), 所以都没有属性

null 与 undefined 用法的区别

  • null
    • 作为函数的参数, 表示该函数的参数不是对象
    • 作为对象原型链的终点[:不理解]
  • undefined
    • 声明被定义了, 但还没赋值时就等于 undefined
    • 调用函数所用到的参数没有提供, 该参数等于 undefined.
    • 对象没有赋值的属性,该属性就是 undefined
    • 函数没有返回值时, 默认返回 undefined

为什么 JS 要同时存在 undefined 和 null?

因为 JS 作为一门动态类型语言, 成员除了表示存在的空值外, 还有可能根本不存在(因为存不存在只在运行期才知道).

复杂数据

  • object 由属性和属性的值构成的无序列表的合集

相较于基础数据类型的区别

  • 对象有属性
  • 对象有方法, 方法本身是一个特殊的属性
  • 对象可以改变

对象分类

  • 内部对象
    • 错误对象
    • 常用对象
      • 字符串
      • 数字
      • object
      • fn
      • RegEx
      • boolean
      • 日期
    • 内置对象, 如何需用 new 操作符
      • global
      • JSON
      • Math
  • 宿主对象( window)

对对象进行比较通常的目的在于对对象的引用进行比较.

类型转换

  • 原始类型 -> string 的转换
    • "x"
    • x + ""
    • String(x)
  • 原始类型 -> number
    • Number(undefined) -> NaN;
    • Number(null) -> 0;
    • Number(string)
      • 如果 string 中就是数字, 则去掉双引号. 否则就是 NaN
      • 若字符串是空的, 则是 0;
  • 原始类型 -> 布尔值
    • !!x
    • Boolean(x)

  • 原始类型 -> 对象

    • 只有 null undefined 不能转换为对象

  • 对象 -> 原始类型

    • 布尔, 都是 true
    • 数字
      1. valueOf(), 返回是原始值则返回原始值
      2. toString()
    • 字符串
      1. toString()
      2. valueOf()

toString(x)可以接受参数 x = 2~36, 当想要随机生成字母和数字混排时. Math.random(toString(36))[:有问题,并不能生成随机字母].

使用 var 创造变量与不适用 var 的区别

  • var 创造的是局部变量, 不使用 var 则是全局变量; 如果你 re-declare 一个变量, 他不会丢失它的值[:什么意思?].

原理: 当 var 定义一个变量, 它的作用域就是当前的函数.

  • var 创造的变量不可以删除, 全局变量是 configurable property 可设置的属性,因此也就可以被删除;

原理: 当 var 定义了一个变量, 而变量都有一个作用域, 为当前的函数;由于全局变量的本质是给 windows 增加一个属性 windows.a;.当不使用 var, 而是给 windows 增加了一个属性, 由于对象上的属性是可以被删除, 因此该全局变量会被删除.而由于 windows 作为一个对象是无序合集的,因此也就不存在提前不提前了.
tag:question

  • var 定义的变量会被提前(are created before any code is executed), 未定义变量只有在赋值给他们的代码执行时才存在.

补充知识: var hoisting 变量提前

变量声明被提前的含义是指声明变量在其他代码执行之前创建, 即 JS 函数中所有变量的声明都被”提前”至函数的顶部, 但是变量赋值是不会的.

当变量声明和赋值在一起使用时, JS 引擎会自动将它分为两部分以便将变量声明提前, 不将赋值过程提前是因为有可能影响代码执行出不可预期的结果.

第一个 alert(scope) 为 undefined 的原因在于,函数体内的变量声明”提前”至函数顶端, 而变量的初始化却留在了远处.因此应该将变量的声明放在作用域顶端,从而清楚区分变量的作用域.

若是全局类似下得出 undefined, 则原因是因为 window.varible 的初始值就是 undefined.

由于函数作用域的特性, 局部变量在整个函数体内都是有定义的, 因此若存在相同的全局变量会将其覆盖并初始化

操作符

由于各操作符之间可能存在的隐式类型转换, 因此若要进行全等比较最好使用 ===.当不知道两个数据类型的操作结果(如加减乘除)是否能获得你想要的结果时, 最好先用显示类型转换然后再相加.

一元操作符

操作符之间的处理原则

  1. 优先级
    1. 属性访问(使用 . 或 ())表达式
    2. 一元操作符在所有的操作中拥有最高的优先级
    3. 赋值表达式最低
  2. 结合性
    • 绝大部分操作是左结合
    • 只有一元操作符, 三元操作符, 赋值运算符是右结合, 即从右到左算.比如 a ? b : c ? d : e = (a ? b : (c ? d : e))
  3. 如果表达式中存在子表达式, 它们的运算顺序都是从左向右.

三个特殊的一元操作符:

  1. typeof 返回表达式的数据类型, 返回的数据格式是 string
    • “undefined”: 未声明和未初始化的变量
    • “boolean”
    • “number”
    • “object”
      • 对象
      • null
    • “function” 因此函数具有一些特殊的属性, 所以需要作出区分
  2. void 返回一个 undefined, 通常用于禁止 <a> 跳转;
  3. delete 删除属性,The delete operator removes a property from an object;

二元操作符

加法操作符, 它存在的两种主要形式

  1. 对数字的加法操作
  2. 两个字符串的拼接
    加法操作符更侧重于字符串的拼接, 因为 "我" + 1 // "我1", 若存在一个字符串,优先将另一个数据转换为字符串.
  • 如果有一个是字符串,则把另外一个也转化成字符串.
  • 若有 undefined | null | boolean | number 混合运算时, 所有东西都会被转换成数字

逻辑非操作符

无论 ! 后的值是什么数据类型, 这个操作符都会返回一个布尔值. 操作过程是先将其操作数转换为一个布尔值, 然后再对其求反. 实施效果等同于Boolean()函数

要注意的是所有数据只有六个值转化为布尔值是 false

所有的对象(包括空的对象)转换成布尔类型都是 true;因为逻辑非属于一元运算符, 拥有最高等级的优先权, 然后 && > || > a ? b : c;

通过 !! 可以将数据类型转换成布尔类型;

逻辑与操作符

短路写法, 如果 a == 1 成立, 则执行后面的内容

与操作符的返回值,运行到哪个表达式,返回值就是对应表达式的返回值.且返回值不一定是布尔类型的;

逻辑或操作符

x = a || b 如果第一个值为 true 则返回第一个值否则就返回 b

a && b || c 如果 a (判断)则执行 b 否则就是 c; 三目运算符

乘性操作符, 求模中小数与小数之间所得结果仍是小数

关系操作符(> < >= <=)

如果有一个是数字, 则把另外一个也转化成数字类型;
二元运算符. 左结合. 下面一个坑 7 > 6 > 5 = var (7 > 6) => true -> 1 > 5 所以返回是 false 这就是左结合的含义.

返回的一定是布尔值;
